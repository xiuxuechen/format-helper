# Word 格式规则模型参考

本 skill 的 `RULE_SUMMARY_TEMPLATE.md` 按 Word 文档的可视结构和底层模型共同组织。

## 参考依据

- WordprocessingML 的主体结构由文档、正文、段落、文本运行和文本组成；典型文档还包含页眉、页脚、脚注、尾注、批注、样式定义和设置等部件。
- WordprocessingML 样式用于集中管理段落、字符、表格、编号等格式，样式与直接格式共同决定最终显示效果。
- Word 文档按节管理页面设置；节包含页边距、方向、页眉页脚等页面布局属性。
- 页眉页脚按节应用，并可区分首页、奇偶页和普通页。
- 段落格式包含对齐、缩进、行距、段前段后等属性；字符格式包含字体、字号、加粗、斜体、下划线、颜色等属性。
- 编号和列表通过编号定义和编号层级表达，标题编号也可能通过样式关联编号定义。
- 表格由表格、行、单元格组成，表格属性、行属性、单元格属性和单元格内段落共同决定视觉效果。

## RULE_SUMMARY.md 排序原则

按用户打开 Word 后最先看到、最容易判断的顺序排序：

1. 文档结构总览
2. 页面与节
3. 页眉页脚
4. 封面与题名
5. 目录
6. 章节标题
7. 正文段落
8. 编号与列表
9. 表格
10. 图片、脚注、批注与修订
11. 样式与直接格式
12. 自动处理边界和规则缺口

## 覆盖范围

当前模板覆盖正式 Word 材料中 90% 以上常见可视格式面：

- 文档部件：主体、样式、编号、设置、页眉、页脚、脚注、尾注、批注、媒体。
- 页面节：纸张、方向、页边距、页眉页脚距离、装订线、分栏、文档网格、文字方向、垂直对齐。
- 页眉页脚：首页、奇偶页、普通页、继承关系、页码字段。
- 段落：对齐、首行缩进、左右缩进、悬挂缩进、行距、段前段后、制表位、分页控制。
- 字符：字体、字号、加粗、斜体、下划线、删除线、颜色、高亮、上标下标。
- 编号：章节编号、正文编号、项目符号、多级列表、连续性。
- 表格：宽度、列网格、行列数量、边框、表头、单元格、合并、跨页、横向页、嵌套表格。
- 特殊内容：图片、图题表题、超链接、字段、脚注尾注、批注、修订记录。

不承诺覆盖所有 OOXML 冷门元素；无法稳定抽取或风险较高的项目必须进入人工确认或专项处理。

## 输出要求

- 给用户看的规则必须是中文可读规则，不是内部样式名。
- 能提取具体值时写具体值，例如字号、页边距、行距。
- 不能稳定提取时写“未能从标准文件稳定提取，需人工确认”。
